# 定时推送指南

## 推荐方式：GitHub Actions 调用已部署服务

GitHub Actions 不负责长期运行网页服务。它只在指定时间调用已经部署并在线的 Visual Radar。

项目内 `.github/workflows/daily.yml` 默认：

- 周一至周五执行
- UTC 01:30
- 新加坡、北京时间 09:30
- 支持在 Actions 页面手动运行
- 手动运行默认 `dry_run=true`，不会发群消息

GitHub 的 `schedule` 使用 UTC cron，且工作流必须存在于默认分支。官方参考：[工作流触发事件](https://docs.github.com/en/actions/reference/workflows-and-actions/events-that-trigger-workflows)。

### 修改时间

例如每天新加坡时间 10:00，相当于 UTC 02:00：

```yaml
schedule:
  - cron: "0 2 * * *"
```

例如每天执行，包括周末：

```yaml
schedule:
  - cron: "30 1 * * *"
```

## 配置步骤

1. 先完成公网部署并确认 `/api/health` 返回成功。
2. 在 GitHub Actions Secrets 添加 `DEPLOYED_URL`。
3. 添加与部署环境一致的 `CRON_SECRET`。
4. 打开 `Actions` → `Visual Radar Daily` → `Run workflow`。
5. 第一次保持干运行，确认日报生成正常。
6. 第二次关闭干运行，确认企业微信群收到消息。

## 备选方式：macOS launchd

适合暂时不部署公网、只在本机自动运行。电脑必须在执行时间开机、联网，Visual Radar 服务也必须运行。

1. 复制模板：

```bash
cp ops/com.visualradar.daily.plist.example ~/Library/LaunchAgents/com.visualradar.daily.plist
```

2. 将模板中的 `/绝对路径/Visual Radar` 改为真实路径。
3. 确认 `pnpm` 路径：

```bash
which pnpm
```

4. 加载任务：

```bash
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.visualradar.daily.plist
```

5. 手动测试：

```bash
launchctl kickstart -k gui/$(id -u)/com.visualradar.daily
```

日志位于 `/tmp/visual-radar-daily.log` 和 `/tmp/visual-radar-daily-error.log`。

## 自动流程失败时

自动化按照“采集 → AI 解读 → 日报生成 → 微信发送”执行。任何一步失败，后续步骤都会停止，不会发送不完整日报。优先检查部署日志、OpenAI 账户 API 权限、持久化数据目录和企业微信 Webhook。

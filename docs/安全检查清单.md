# 安全检查清单

## 每次上传 GitHub 前

- [ ] 仓库中没有 `.env.example` 以外的 `.env*` 文件
- [ ] 没有 OpenAI API Key、企业微信 Webhook、真实 `CRON_SECRET`、token、Cookie 或会话文件
- [ ] 没有本机用户绝对路径
- [ ] `git diff --cached` 已人工检查
- [ ] 已确认 Public 仓库和 Pages JSON 会公开完整 issue 数据

查找所有本地环境文件，排除示例、依赖、构建产物和 Git 元数据：

```bash
find . \
  -path './.git' -prune -o \
  -path './node_modules' -prune -o \
  -path './dist' -prune -o \
  -name '.env*' ! -name '.env.example' -print
```

扫描企业微信地址、常见平台 token、Bearer token，以及敏感变量赋值：

```bash
SECRET_PATTERN='qyapi\.weixin\.qq\.com/cgi-bin/webhook/'\
'send\?key=|sk-[A-Za-z0-9_-]{16,}|github_pat_[A-Za-z0-9_]{16,}|ghp_[A-Za-z0-9]{16,}|Bearer[[:space:]]+[A-Za-z0-9._~+/-]{12,}|(API_KEY|WEBHOOK|SECRET|TOKEN|COOKIE)[[:space:]]*[:=][[:space:]]*[^[:space:]#]{8,}'
rg --hidden -n -i "$SECRET_PATTERN" . \
  -g '!.git/**' -g '!node_modules/**' -g '!dist/**'
```

这些命令只是辅助检查，不是没有秘密的绝对证明。必须人工检查所有命中、暂存 diff、提交文件列表和 Public data 内容。

Public 数据包括仓库中的 `data/visual_radar_issues.json` 和 Pages `public-data/issues/*.json`。完整 issue 可能包含 source URL、原始 text、content hash、score/score breakdown、selection rationale、trend keywords、skipped item IDs 和跳过原因，不只是网页当前渲染的字段。

## 每次正式推送前

- [ ] `VISUAL_RADAR_PUBLIC_URL` 已替换为 Actions deployment 输出，不是占位符
- [ ] Pages Action 已成功，手机可以打开当期详情页
- [ ] 顶部微信精选为 10 条或更少
- [ ] 已运行 `pnpm wecom:preview -- 2026-07-16` 并确认 `sent: false`
- [ ] 用户已明确确认正式发送，群内没有同一期重复消息

未完成任一项时不得发送企业微信。

## 密钥泄露处理

1. 立即禁用或轮换旧密钥。
2. 企业微信删除旧机器人或重新生成 Webhook。
3. 清理 Git 历史不能代替轮换；历史提交可能已被复制。
4. 检查 GitHub Actions 日志和部署日志。
5. 当前 Pages-only 流程没有 GitHub Actions Secrets；只有未来恢复 Express automation 时，才重新配置所需 secrets，并重新审查权限、轮换和工作流边界。

GitHub 官方明确建议删除并轮换暴露的密钥，同时对工作流使用最小权限：[GitHub Actions 安全使用指南](https://docs.github.com/en/actions/reference/security/secure-use)。

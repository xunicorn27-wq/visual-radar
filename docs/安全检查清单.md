# 安全检查清单

## 每次上传 GitHub 前

- [ ] 仓库中没有 `.env` 或 `.env.local`
- [ ] 没有 OpenAI API Key
- [ ] 没有企业微信 Webhook 完整地址
- [ ] 没有真实 `CRON_SECRET`
- [ ] 没有 Cookie、浏览器登录状态或会话文件
- [ ] 没有本机用户绝对路径
- [ ] `git diff --cached` 已人工检查
- [ ] 仓库默认设为 Private

检查命令：

```bash
find . -name '.env' -o -name '.env.local'
rg -n "OPENAI_API_KEY=.+|WECOM_BOT_WEBHOOK=.+|CRON_SECRET=.+" . \
  -g '!node_modules/**' -g '!dist/**' -g '!.env.example'
```

## 每次正式推送前

- [ ] 网页日报内容和图片已检查
- [ ] 顶部微信精选为 10 条或更少
- [ ] `VISUAL_RADAR_PUBLIC_URL` 可以从手机访问
- [ ] 先执行过 `--dry-run`
- [ ] 群内没有同一期重复消息

## 密钥泄露处理

1. 立即禁用或轮换旧密钥。
2. 企业微信删除旧机器人或重新生成 Webhook。
3. 清理 Git 历史不能代替轮换；历史提交可能已被复制。
4. 检查 GitHub Actions 日志和部署日志。
5. 重新配置部署环境和 GitHub Secrets。

GitHub 官方明确建议删除并轮换暴露的密钥，同时对工作流使用最小权限：[GitHub Actions 安全使用指南](https://docs.github.com/en/actions/reference/security/secure-use)。

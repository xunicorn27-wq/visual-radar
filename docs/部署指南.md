# 部署指南

## 当前发布方式：GitHub Pages

当前采用 Pages-only/Codex 模式：Codex 在本地协助采集、Agent 分析和日报生成，确认后的静态数据推送到 Git，GitHub Actions 只构建并发布 GitHub Pages。

Pages 地址格式为 `https://YOUR-GITHUB-USER.github.io/visual-radar`。`YOUR-GITHUB-USER` 是必须替换的占位符；实际值以 GitHub Actions deployment 输出为准。仓库需设为 Public，并在 `Settings` -> `Pages` 中选择 `Source: GitHub Actions`。

当前 Pages 发布：

- 不需要公网 Express 服务。
- 不需要 OpenAI API Key。
- 不需要任何 GitHub Actions Secrets。
- 不要在 GitHub Actions 添加 `DEPLOYED_URL` 或 `CRON_SECRET`。

Pages 不运行采集、分析、日报生成、daily automation 或企业微信发送。需要动态管理接口时，可在本地运行 Express；这不改变 Pages 的静态发布边界。

`OPENAI_API_KEY` 为空不影响 Codex Agent 模式。企业微信 Webhook 和 `CRON_SECRET` 只保存在本地 `.env.local`；Pages 发布成功并完成公网验证后，仍需先运行 `pnpm wecom:preview -- 2026-07-16`，并等待用户明确确认才可正式发送。占位 URL 未替换或未验证时不得发送。

## 可选未来部署：公网 Express

以下 Docker 和 HTTPS 配置仅适用于未来确实需要长期在线的 Express 服务时，不是当前 Pages 发布的前置条件。恢复公网 Express daily automation 前还必须完成新的安全评审。

### 部署要求

- 支持 Node.js 22 或 Docker
- 可配置环境变量
- 可挂载持久化磁盘或目录
- 可以长期在线并通过 HTTPS 公网访问

不要部署到只有临时文件系统且没有持久化存储的服务，否则重启后新日报可能丢失。

### Docker Compose 部署

服务器安装 Docker 后：

```bash
git clone https://github.com/你的账号/visual-radar.git
cd visual-radar
cp .env.example .env.local
```

编辑 `.env.local`，至少设置：

```dotenv
OPENAI_API_KEY=
OPENAI_MODEL=gpt-5.4-mini
WECOM_BOT_WEBHOOK=
VISUAL_RADAR_PUBLIC_URL=https://YOUR-EXPRESS-DOMAIN.example
CRON_SECRET=
VISUAL_RADAR_DATA_DIR=/app/data
PORT=3099
```

启动：

```bash
docker compose up --build -d
docker compose logs -f
```

Docker Compose 将本机 `data/` 映射到容器 `/app/data`。Docker 官方说明可通过卷或绑定挂载持久化容器数据：[Node.js 容器化指南](https://docs.docker.com/guides/nodejs/)、[Dockerfile VOLUME](https://docs.docker.com/reference/dockerfile/#volume)。

### 反向代理与 HTTPS

生产环境建议使用托管平台自带域名，或通过 Caddy、Nginx、Cloudflare 等把 HTTPS 域名转发到容器的 3099 端口。

完成后检查：

```bash
curl https://YOUR-EXPRESS-DOMAIN.example/api/health
```

预期：

```json
{"ok":true,"service":"visual-radar"}
```

### 环境变量放在哪里

- 当前 Pages-only/Codex 模式：GitHub Actions 不需要任何 Secrets。
- 可选公网 Express：OpenAI、企业微信和服务端 `CRON_SECRET` 放在部署平台的 Secret/Environment 设置。
- 只有未来恢复 GitHub Actions daily automation 并重新安全评审后，触发工作流才可能需要 `DEPLOYED_URL` 和匹配的 `CRON_SECRET`。
- 不要把 `.env.local` 上传到仓库。

OpenAI API 使用 API 平台创建的密钥，并通过服务端环境变量读取。ChatGPT 登录状态或公司 ChatGPT 工作区登录不能代替 API 计费与权限。官方参考：[OpenAI API 快速开始](https://platform.openai.com/docs/quickstart)、[模型目录](https://developers.openai.com/api/docs/models)。

### 更新可选 Express 部署

```bash
git pull
docker compose up --build -d
```

更新前备份 `data/`：

```bash
cp -R data "data-backup-$(date +%Y%m%d-%H%M%S)"
```

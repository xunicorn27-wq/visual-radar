# 企业微信推送指南

## 1. 创建群机器人

在目标企业微信群中创建群机器人并复制 Webhook。Webhook 相当于群消息发送密码，不要发到聊天中、截图或写入 GitHub。

官方参考：[企业微信群机器人配置说明](https://developer.work.weixin.qq.com/document/path/91770)。

## 2. 只在本机配置

编辑未提交 Git 的本地 `.env.local`：

```dotenv
WECOM_BOT_WEBHOOK=
VISUAL_RADAR_PUBLIC_URL=https://YOUR-GITHUB-USER.github.io/visual-radar
CRON_SECRET=
```

离线预览只需要正确的 `VISUAL_RADAR_PUBLIC_URL`。其中占位符必须替换为 GitHub Actions deployment 输出的实际 Pages URL，并去掉结尾斜杠。`WECOM_BOT_WEBHOOK` 和 `CRON_SECRET` 只在用户确认后的正式发送或本地管理操作中需要。

Pages 不读取这些秘密，也不会发送消息。未替换 Pages URL、Actions 未成功或公网详情页未验证时，不得发送企业微信。

## 3. 强制门禁

1. push 本期 data 分析和 issues 到 `main`。
2. 等待 GitHub Pages Action 成功，以 deployment 输出确认实际 URL。
3. 用手机打开当期公网详情页，检查内容、图片和链接。
4. 运行单期离线预览并检查 Markdown。
5. 只有用户明确确认后，才可启动本地 Express，并在管理区点击“推送企业微信”，或调用明确的不带 dry-run 的正式接口。

不要使用已删除的 daily workflow，也不要运行 `pnpm daily` 代替上述门禁。

## 4. 单期 dry-run 预览

```bash
pnpm wecom:preview -- 2026-07-16
```

将日期替换为真实存在的本地 issue ID。脚本自行读取项目根目录的 `.env.local` 和 `data/visual_radar_issues.json`，校验 Pages URL 后直接调用 Markdown 构建器，输出 `markdown` 和 `sent: false`。

该命令不需要启动 Express，不读取 `CRON_SECRET`、Webhook 或任何发送函数，也不发起网络请求，因此不会触达群聊。

Markdown 中的详情链接必须使用 Actions deployment URL，例如：

```text
https://YOUR-GITHUB-USER.github.io/visual-radar/issues/2026-07-16
```

这里的占位符必须替换。链接未从手机验证时不得发送。

## 5. 明确确认后正式发送

本指南不提供正式发送 curl 命令。只有用户看过公网页面和 preview Markdown，并明确说可以发送后，才可在本地管理区执行单期发送，或由操作者明确调用不带 dry-run 的正式接口。没有明确确认时必须停在 `wecom:preview`。

群消息只包含 `featuredStoryIds` 标记的最多 10 条精选，Public Pages JSON 仍公开完整 issue 数据。

## 6. 常见问题

- **Webhook 未配置**：检查本地 `.env.local`。
- **日报不存在**：确认指定日期已经写入本地 `data/visual_radar_issues.json`。
- **Pages URL 无效**：确认 `VISUAL_RADAR_PUBLIC_URL` 是不带 query/hash 的 HTTP 或 HTTPS 地址。
- **阅读全文打不开**：以 Actions deployment 输出为准，替换占位符并去掉结尾斜杠。
- **401 未授权**：本地 `CRON_SECRET` 与 Express 服务不一致。
- **重复发送**：正式发送前检查群内是否已有同一期消息。

如果 Webhook 曾出现在公开仓库、截图或聊天记录中，请删除旧机器人或重新生成 Webhook。

# 企业微信推送指南

## 1. 创建群机器人

在目标企业微信群中打开群设置，找到“群机器人”或“添加群机器人”，创建机器人后复制 Webhook 地址。不同企业的管理员策略可能会隐藏此入口，需要企业微信管理员允许群机器人。

官方参考：[企业微信群机器人配置说明](https://developer.work.weixin.qq.com/document/path/91770)。

Webhook 相当于群消息发送密码。不要发到聊天中，不要截图，不要写入 GitHub。

## 2. 本机配置

编辑 `.env.local`：

```dotenv
WECOM_BOT_WEBHOOK=在这里填写完整的HTTPS地址
VISUAL_RADAR_PUBLIC_URL=https://你的日报公网地址
CRON_SECRET=一段至少32字符的随机字符串
```

`VISUAL_RADAR_PUBLIC_URL` 必须能被企业微信用户访问。本机的 `localhost` 只能自己打开，不能作为正式群推送地址。

## 3. 预览消息

启动系统后，先执行干运行：

```bash
pnpm daily -- --dry-run
```

也可以调用接口：

```bash
curl --request POST \
  --header "x-cron-secret: 你的管理密钥" \
  "http://localhost:3099/api/visual-radar/issues/2026-07-16/send-wecom?dryRun=1"
```

干运行会返回 Markdown，但不会触达群聊。

## 4. 正式发送

网页管理区输入 `CRON_SECRET`，点击“推送企业微信”；或者执行：

```bash
pnpm daily
```

群消息只包含 `featuredStoryIds` 标记的最多 10 条精选，网页仍展示全部高价值中文内容。

## 5. 常见问题

- **Webhook 未配置**：检查变量名和部署平台环境变量。
- **阅读全文打不开**：公网地址错误，或服务只运行在本机。
- **401 未授权**：网页输入的管理密钥与服务端 `CRON_SECRET` 不一致。
- **消息太长**：系统已压缩每条摘要；如果继续增加精选数量，需要重新检查企业微信消息长度限制。
- **重复发送**：GitHub Actions 使用并发锁，但人工点击仍可能重复；正式发送前先看群内是否已有当期消息。

如果 Webhook 曾出现在公开仓库、截图或聊天记录中，请删除旧机器人或重新生成 Webhook。

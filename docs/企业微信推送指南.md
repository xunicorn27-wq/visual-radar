# 企业微信推送指南

## 1. 创建群机器人

在目标企业微信群中打开群设置，找到“群机器人”或“添加群机器人”，创建机器人后复制 Webhook 地址。不同企业的管理员策略可能会隐藏此入口，需要企业微信管理员允许群机器人。

官方参考：[企业微信群机器人配置说明](https://developer.work.weixin.qq.com/document/path/91770)。

Webhook 相当于群消息发送密码。不要发到聊天中，不要截图，不要写入 GitHub。

## 2. 只在本机配置秘密

编辑本地 `.env.local`：

```dotenv
WECOM_BOT_WEBHOOK=在这里填写完整的HTTPS地址
VISUAL_RADAR_PUBLIC_URL=https://visual-radar-owner.github.io/visual-radar
CRON_SECRET=在这里填写至少32字符的随机字符串
```

`VISUAL_RADAR_PUBLIC_URL` 必须是 GitHub Pages 项目地址，并且不带结尾斜杠。`WECOM_BOT_WEBHOOK` 和 `CRON_SECRET` 只能保存在本地 `.env.local`，不得提交到 GitHub，也不需要配置为 GitHub Actions Secrets。

GitHub Pages 只发布静态网页，不运行服务端接口，也不会发送任何企业微信消息。

## 3. 发送前的强制检查

必须按以下顺序操作：

1. push 本期 data 分析和 issues 到 `main`。
2. 等待 GitHub Pages Action 成功。
3. 用手机打开当期公网详情页，确认内容、图片和“阅读全文”链接可访问。
4. 启动本地 Express 服务，调用当期单期 dry-run 预览实际 Markdown。
5. 将预览结果交给用户检查；只有用户明确确认后，才能正式发送。

不要使用已删除的 daily workflow，也不要运行 `pnpm daily` 代替上述检查。

## 4. 单期 dry-run 预览

预览已有日报，不重新采集、分析或生成内容。在受信任的 Visual Radar 项目目录执行以下命令；第一行只把本地、未提交 Git 的 `.env.local` 加载到当前 shell 环境：

```bash
set -a; . ./.env.local; set +a
curl --fail-with-body --silent --show-error \
  --request POST \
  --header "x-cron-secret: $CRON_SECRET" \
  "http://localhost:3099/api/visual-radar/issues/2026-07-16/send-wecom?dryRun=1"
```

将日期替换为当期 issue ID。返回结果应包含 `sent: false` 和 Markdown；该请求不会触达群聊。检查 Markdown 中的详情链接应类似：

```text
https://visual-radar-owner.github.io/visual-radar/issues/2026-07-16
```

群消息只包含 `featuredStoryIds` 标记的最多 10 条精选，网页仍展示全部高价值中文内容。

## 5. 明确确认后正式发送

只有用户在看过公网页面和 dry-run Markdown 后明确说可以发送，才可执行不带 `dryRun=1` 的单期请求：

```bash
set -a; . ./.env.local; set +a
curl --fail-with-body --silent --show-error \
  --request POST \
  --header "x-cron-secret: $CRON_SECRET" \
  "http://localhost:3099/api/visual-radar/issues/2026-07-16/send-wecom"
```

没有明确确认时必须停在 dry-run，不得试发、补发或使用自动化发送。

## 6. 常见问题

- **Webhook 未配置**：检查本地 `.env.local` 的变量名。
- **阅读全文打不开**：确认 Pages Action 已成功，项目地址正确且没有结尾斜杠。
- **401 未授权**：请求头与本地 `CRON_SECRET` 不一致。
- **消息太长**：系统已限制最多 10 条并压缩摘要；不要自行增加精选数量。
- **重复发送**：正式发送前检查群内是否已经存在同一期消息。

如果 Webhook 曾出现在公开仓库、截图或聊天记录中，请删除旧机器人或重新生成 Webhook。
